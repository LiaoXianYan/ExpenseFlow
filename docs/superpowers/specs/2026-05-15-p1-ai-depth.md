## P1-1：RAG 改造为向量检索（已确认）

### 技术栈

| 组件 | 选型 |
|------|------|
| 嵌入模型 | 阿里云 DashScope text-embedding-v2（API Key: sk-357c...） |
| 向量存储 | LangChain4j InMemoryEmbeddingStore |
| LLM | DeepSeek Chat（已配置） |
| 文档分块 | LangChain4j DocumentSplitter（500字/chunk, 50字重叠） |

### 架构流程

```
启动时（@PostConstruct）:
  docs/policies/*.md
    → FileSystemDocumentLoader
    → DocumentSplitter(500 chars, 50 overlap)
    → DashScopeEmbeddingModel.embedAll(chunks)
    → InMemoryEmbeddingStore.add(embeddings)

查询时（RagService.ask）:
  用户问题
    → DashScopeEmbeddingModel.embed(question)
    → EmbeddingStore.search(question, maxResults=3)
    → 检索到 top-3 最相关 chunk
    → Prompt: "根据以下政策回答：{chunk1}\n{chunk2}\n{chunk3}\n\n问题：{question}"
    → DeepSeek Chat.generate(prompt)
    → RagAnswerVO
```

### 改动范围

| 文件 | 操作 |
|------|------|
| `ai-service/pom.xml` | 加 langchain4j-embeddings 依赖 |
| `ai-service/.../config/LangChain4jConfig.java` | 加 DashScopeEmbeddingModel Bean |
| `ai-service/.../service/impl/RagServiceImpl.java` | 重写为向量检索 |
| `docs/policies/travel-policy.md` | 新建差旅政策文档 |
| `docs/policies/expense-policy.md` | 新建报销政策文档 |
| `ai-service/.../service/RagServiceImplTest.java` | 新建测试 |

### 容错

- Embedding API 异常 → 降级为纯 LLM 回答（用原 hardcoded policyContext）
- LLM API 异常 → 返回固定提示文案

### 测试

1. 嵌入 + 检索 返回最相关 chunk
2. RAG 回答引用政策内容
3. 嵌入 API 失败 → 降级
