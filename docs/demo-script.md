# ExpenseFlow 演示脚本

## 准备工作

1. 启动全部服务 (双击 `start-all.bat` 或 Docker Compose)
2. 打开浏览器 → `http://localhost:5173`
3. 登录账号: `admin` / `password`

## 演示流程

### 第一步：登录 + Dashboard (30秒)

- 输入 `admin` / `password` → 登录
- Dashboard 展示：统计卡片 (出差申请/报销单/待审批/打款金额)
- ECharts 饼图 (费用分布) + 柱状图 (审批统计)

### 第二步：出差申请 (1.5分钟)

1. 侧边栏 → 出差申请 → 新建申请
2. 填写：事由"客户拜访"、目的地"上海"、日期、预估金额 6000 元
3. 保存 → 提交审批
4. 状态变为 APPROVING

### 第三步：审批流转 (1.5分钟)

1. 侧边栏 → 审批工作台
2. 选择"经理审批"标签 → 看到待办任务
3. 点击"通过" → 输入意见"同意" → 确定
4. 切换到"总监审批"标签 → 出现新任务 (金额>5000 需总监审批)
5. 点击"通过" → 确定
6. 回到出差申请列表 → 状态变为 APPROVED

### 第四步：报销单 (2分钟)

1. 侧边栏 → 报销管理 → 新建报销单
2. 填写报销日期 → 保存
3. 添加明细：交通费 3000 (高铁) + 住宿费 2500
4. 提交审批 → 状态变为 APPROVING
5. 审批工作台 → 财务审核 → 通过
6. 审批工作台 → 经理审批 → 通过
7. 报销列表 → 状态变为 APPROVED
8. 点击"打款" → 状态变为 PAID

### 第五步：发票上传 + OCR (1分钟)

1. 侧边栏 → 发票管理
2. 上传一张发票图片 (PNG/JPG)
3. 点击"OCR 识别" → 显示识别结果

### 第六步：AI 审单 + 智能助手 (1分钟)

1. 侧边栏 → AI 能力 → AI 审单
2. 选择业务类型"报销单"、输入金额和明细
3. 点击"AI 审单" → 显示审单结果和风险等级
4. 切换到 AI 助手
5. 输入："住宿费报销标准是多少？"
6. AI 回答政策内容

### 第七步：消息通知 (30秒)

1. 侧边栏 → 消息通知
2. 查看审批结果通知消息
3. 标记已读 / 全部已读

## 演示要点

- **微服务架构**: 侧边栏各模块由不同微服务提供
- **工作流**: 出差申请/报销单的状态流转由 Flowable 引擎驱动
- **规则引擎**: 金额>5000 自动触发总监审批
- **AI 集成**: LangChain4j + DeepSeek 实现智能审单
- **异步消息**: RabbitMQ 实现服务间解耦通信
- **多租户**: 所有数据通过 tenant_id 隔离

## 时间控制

总计约 8 分钟演示时间
